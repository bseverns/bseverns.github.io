A few active routes into the current work

active instrument

MOARkNOBS-42

Controller platform work centered on mappings, latency, and legible instrument behavior.

Plainly: a custom MIDI controller whose behavior can be tested, read, and taught.

Proof line: controller mappings, schema, and latency-lab documentation.

workshop system

Human-Buffer

Workshop structure for computer vision, consent, and public understanding of what a system perceives.

Plainly: a workshop that helps people see what a camera system notices and what it should not keep.

Proof line: vision exercises, consent framing, and public-facing explanation.

Proof status: checking source trail.

machine-docs

Runbook and field-manual layer where machine maintenance becomes shared curriculum.

Plainly: practical machine notes that turn maintenance into something others can learn.

Proof line: repair notes, operating context, and shared maintenance language.

Proof status: checking source trail.

frZone

Frequency-zone trigger study connecting sound analysis to MIDI and OSC events.

Proof line: analysis thresholds, triggers, and routing behavior.

Proof status: checking source trail.
